#### 继承
在定义一个新的类B时,如果该类与某个已有的类A相似 (指的是B拥有A的全部特点)
我们称 A—基类(父类),B—基类的派生类(子类) ==“B是A”==
派生类 是通过对基类进行修改 和扩充 得到的 
- 扩充:在派生类中,可以添加新的成员变量和成员函数 
- 修改:在派生类中,可以重新编写从基类继承得到的成员 
派生类一经定义后,可以独立使用,不依赖于基类
==这样共性的东西不用单独处理，对于子类只要额外增加他自身的特性就可以了==

写法
```
class B:派生方式说明符 A
{
B自身具有的成员属性和成员方法
};
```
派生类拥有基类的全部成员,包括
- private/protected/public成员变量
- private/protected/public方法 
在派生类的各个成员函数中,不能访问基类中的 private成员,因此要在父类中提供额外的接口
子类中与父类同名函数会覆盖父类函数
###### 派生类对象的内存空间
派生类对象的大小,等于基类对象的大小 + 派生类对 象自己的成员变量的大小 
在派生类对象中, 包含着基类对象, 而且基类对象的存储位置位于派生类对象新增的成员变量之前

###### 复合关系与继承关系
继承: "是" 关系 
- 基类 A 
- B是基类A的派生类 
- 逻辑上要求："一个B对象也是一个A对象" 
复合: "有" 关系 
- 复合, 即一个类的对象拥有作为其成员的其它类的对象 
- d是类C的成员 
- d是类D的一个对象 
- 复合关系满足：C类中 "有" 成员对象d
###### 复合关系的使用
当一个A类中具有B类作为成员变量，B类中又有A类具有成员变量
此时不能简单地在每个类中创建另一个类的变量，因为这样会导致递归调用两个类的构造函数，陷入死循环
==避免循环定义的方式: 在一个类中使用另一个类的指针, 而不是对象作为成员变量==

```
class CDog; //注意要提前声明
class CMaster { CDog * dogs[10]; }; 
class CDog { CMaster * m; };
```
###### 隐藏
派生类可以定义一个和基类成员同名，同参数表的成员,这叫隐藏
在派生类中访问这类成员时,默认的情况是访问派生类中定义的成员 
要在派生类中访问由基类定义的同名成员时,要使用作用域符号 `父类名::成员名`
==一般地,基类和派生类 不定义同名成员变量==
###### 权限访问
基类的private成员: 可以被下列函数访问 
- 基类的成员函数 
- 基类的友员函数 
基类的public成员: 可以被下列函数访问 
- 基类的成员函数 
- 基类的友员函数 
- 派生类的成员函数 
- 派生类的友员函数 
- 其他的函数
基类的protected成员: 可以被下列函数访问 
- 基类的成员函数 
- 基类的友员函数 
- 派生类的成员函数可以访问当前对象的基类的保护成员 
- this指针指向的对象 
- 也可以访问其它同类对象的基类的保护成员 
- 派生类的友元函数也可以访问基类的保护成员
- 可访问范围比private成员大, 比public成员小 

| 访问位置      | 卧室(private) | 书房(protected) | 客厅(public) |
| --------- | ----------- | ------------- | ---------- |
| Father自己  | ✅           | ✅             | ✅          |
| Son（在自己家） | ❌           | ✅             | ✅          |
| Son（在兄弟家） | ❌           | ✅             | ✅          |
| Son（在别人家） | ❌           | ❌             | ✅          |
###### 三种继承方式
公有继承 (public) 
- 基类的公有成员和保护成员作为派生类的成员时, 它们都保持原有的状态, 而基类的私有成员仍然是私有的, 不能被这个派生类的子类所访问 
私有继承 (private) 
- 基类的公有成员和保护成员都作为派生类的私有成员, 并且不能被这个派生类的子类所访问 
保护继承 (protected) 
- 基类的所有公有成员和保护成员都成为派生类的保护成员, 并且只能被它的派生类成员函数或友元访问, 基类的私有成员仍然是私有的

| 继承方式 \ 基类成员 | public    | protected | private |
| ----------- | --------- | --------- | ------- |
| 公有继承        | public    | protected | 不可见     |
| 私有继承        | private   | private   | 不可见     |
| 保护继承        | protected | protected | 不可见     |
也就是原来权限和继承方式取最小
#### 派生类的构造和析构函数
在创建派生类的对象时, 需要调用基类的构造函数 
- 初始化派生类对象中从基类继承的成员 
- 在执行派生类的构造函数之前, 总是先执行基类的构造函数
调用基类构造函数的两种方式 
- 显式方式: 在派生类的构造函数中, 为基类构造函数提供参数 。即类似于初始化列表，调用基类的构造函数
```
derived(arg_derived-list):base(arg_base-list) 
```
- 隐式方式: 在派生类的构造函数中, 省略基类构造函数时, 派生类的构造函数则自动调用基类的默认（无参）构造函数。所以要么基类有默认（无参）构造（可以自己重写），则可以不调用基类的无参构造，否则就必须调用基类的构造函数，否则会编译错误
派生类的析构函数被执行时, 执行完派生类的析构函数后, 自动调用基 类的析构函数。编译器会自动调用，不用手动实现

==特殊：拷贝构造函数==
**派生类的拷贝构造函数需要负责调用基类的拷贝构造函数**，而不能依赖基类的默认构造函数。
当你编写派生类的拷贝构造函数时，如果不显式调用基类的拷贝构造函数，编译器会自动调用基类的**默认构造函数**（无参构造），而不是基类的拷贝构造函数。
这意味着：
 - 基类的成员会被默认初始化，而不是从源对象中拷贝过来
 - 最终派生类对象中，基类部分的值不会被正确复制，导致**部分拷贝**（基类部分丢失原有数据）
==由于派生类对象可以隐式转换为基类引用（派生类对象包含基类部分），所以直接将源派生类对象传给基类拷贝构造函数即可。编译器会自动提取其中的基类部分进行拷贝。==
因此，正确的做法是在派生类拷贝构造函数的初始化列表中，**显式调用基类的拷贝构造函数**，将源对象中的基类部分传递给基类进行拷贝。
示例：
```
class Base {
public:
    int a;
    Base(int x = 0) : a(x) { }
    
    // 基类拷贝构造函数（编译器生成的或显式定义的）
    Base(const Base& other) : a(other.a) {
        cout << "Base copy" << endl;
    }
};

class Derived : public Base {
public:
    int b;
    Derived(int x = 0, int y = 0) : Base(x), b(y) { }
    
    // 派生类拷贝构造函数
    Derived(const Derived& other) 
        : Base(other),    // 传入 other，提取其中的基类部分
          b(other.b) {
        cout << "Derived copy" << endl;
    }
};

int main() {
    Derived d1(10, 20);
    Derived d2(d1);  // 调用拷贝构造
    
    cout << d2.a << endl;  // 10
    cout << d2.b << endl;  // 20
    
    return 0;
}
```
注意
1. **传入的是源对象本身**：`Base(other)` 中，`other` 是 `Derived` 类型，但编译器会将其作为 `Base` 的引用传递给基类拷贝构造函数，只拷贝基类部分。
2. **不能传入 `other.Base`**：语法错误，不能直接访问基类子对象的名字。
3. **不能只传部分**：必须传入完整的派生类对象引用，让基类自己决定如何拷贝自己的部分。
4. **如果有多个基类**：每个基类都需要在初始化列表中显式调用各自的拷贝构造函数，传入相同的源对象。


创建派生类的对象时, 执行一个派生类的构造函数之前: 
- 调用基类的构造函数 初始化派生类对象中从基类继承的成员 
- 调用成员对象类的构造函数 初始化派生类对象中成员对象 
派生类析构函数被执行时, 执行完派生类的析构函数后: 
- 调用成员对象类的析构函数 
- 调用基类的析构函数 
析构函数的调用顺序与构造函数的调用顺序相反
例如
```
class Base { 
public: 
int n; 
Base(int i):n(i) { cout << "Base " << n << " constructed" << endl; } 
~Base() { cout << "Base " << n << " destructed" << endl; }
 }; 
 
 class Derived:public Base { 
 public: 
 Derived(int i):Base(i) { cout << "Derived constructed" << endl; } 
 ~Derived() { cout << "Derived destructed" << endl; } };
```

###### public继承的赋值兼容规则
派生类的对象可以赋值给基类对象 
- b = d; 
- 该对象不包含派生类定义 的成员, 因为没有派生类 成员的存储空间 (派生类 部分在赋值时被“切掉”),即成员的切割
- 根本原因是b的内存大小固定了
派生类对象可以初始化基类引用 
- base & br = d; 
派生类对象的地址可以赋值给基类指针 
- base * pb = & d; 
如果派生方式是 private 或 protected, 则上述三条不可行
#### 派生类与基类的指针类型转换
公有派生的情况下, 派生类对象的地址可以直接赋值给基类指针 
`Base * ptrBase = & objDerived;` 
- ptrBase指向的是一个Derived类的对象 
- \*ptrBase可以看作一个Base类的对象, 访问它的public成员 
- 直接通过ptrBase, 不能够访问objDerived由Derived类扩展的成员 
- 即便基类指针指向的是一个派生类的对象, 也不能通过基类指针访问基类没有, 而派生类中有的成员
- 指针访问基类和派生类同名的成员，访问的还是基类的
- 通过强制指针类型转换 可以把ptrBase转换成Derived类的指针,但是程序员要保证ptrBase指向的是一个Derived类的对象, 否则很容易会出错
```
Base * ptrBase = & objDerived; 
Derived *ptrDerived = (Derived * ) ptrBase;
```
==根据指针/引用类型来 决定调用的函数==
#### 直接基类与间接基类
类A派生类B, 类B可再派生类C, 类C派生类D, ... 
- 类A是类B的直接基类 
- 类B是类C的直接基类, 类A是类C的间接基类 
- 类C是类D的直接基类, 类A, B是类D的间接基类
在声明派生类时, 派生类的首部只需要列出它的直接基类 
- 派生类的首部不要列出它的间接基类 
- 派生类的构造函数只要调用直接基类的构造函数即可
- 派生类沿着类的层次自动向上继承它的间接基类 
- 派生类的成员包括 派生类自己定义的成员，直接基类中定义的成员，间接基类的全部成员
#### 多继承
一个类可以从多个基类派生而来, 以继承多个基类的成员。这种派生称作 "多重继承" 
```
class derived:access-specifier 1 base1 , access-specifier 2 base2 , ... { ... };
```
access-specifieri可以是private, protected, public之一

多重继承中, 派生类对象创建时, 先按继承顺序调用基类的构造函数, 然后再调用派生类的构造函数。如果是显示调用，那么在派生类的构造函数中要显式调用所有基类的构造函数 

如果派生类是封闭类, 那么成员对象的构造函数在基类的构 造函数调用结束后依次调用, 最后才调用派生类的构造函数 
多重继承中, 派生类对象的创建过程 
- 按继承顺序调用基类的构造函数 
- 依次调用成员对象的构造函数 
- 调用派生类的构造函数
###### 菱形继承的二义性
1. 当两个中间基类具有同名的成员时，直接在派生类中调用这个成员会引发歧义，因此如果要调用要使用命名空间，即：：来标注调用的是哪一个基类的
```
class base1 { 
private: 
int b1; 
void set(int i) { b1 = 1; } public: int i;
 }; 

class base2 { 
private:
int b2; 
public: 
void set(int i) { b2 = i; } 
int get() { return b2; } 
int i; 
};

class derived: public base1, public base2{
 public: 
 void print() { printf( "%d", get() ); 
 set(5); // 二义性, error 
 base2::set(5) ; // ok 
 base1::set(5); // error, set is private in base1 } };

int main () { 
derived d;
d.set(10); //二义性 
d.base1::set(10); //error, 不能访问私有成员 
d.base2::set(5); 
d.base1::i =5; 
d.base2::i = 5; 
Return 0; 
}
```
注意，二义性检查在访问权限检查之前进行，不能靠成员的访问权限来消除二义性

2. 由于中间的基类都会继承一份父基类中的成员，因此在派生类会继承两份父基类的成员。一方面导致了冗余，另一方面引发二义性，需要使用虚继承让顶层基类在派生类中只保留一份共享的实例。
- 普通继承时，派生类中直接包含基类的全部成员，基类子对象在派生类中的位置是 **固定的、直接的**。
- 虚继承时，派生类不再直接包含虚基类的成员，而是包含一个 **指针（或偏移量信息）**，通过这个指针 **间接访问** 虚基类的成员。
 - **虚基类**（virtual base class）：指的是被虚继承的那个类，也就是最顶层的 `Grand`
- **虚继承**（virtual inheritance）：指的是继承时使用 `virtual` 关键字的方式，如 `class Base1 : virtual public Grand`
- **中间基类（直接虚继承自虚基类的类）中包含指向虚基类的指针或偏移量信息。** 最派生类本身不再额外包含这些指针，而是通过中间基类中已有的指针来访问共享的虚基类。这样最派生类就只有一份虚基类的成员了
内存布局：
Derived 对象中：
Base1 部分：
  - 有自己的成员（如 base1_data）
  - 有一个指针（vbptr），指向 Grand 的位置
Base2 部分：
  - 有自己的成员（如 base2_data）
  - 也有一个指针（vbptr），指向 Grand 的位置
Grand 部分（只有一份）：
  - grand_data（被 Base1 和 Base2 共享）
两个指针指向同一个 Grand，所以数据是相同的、共享的。

注意，如果 `Base1` 和 `Base2` 分别独立实例化对象：
- `Base1 b1;` 创建了一个独立的对象，里面包含 **自己独占的一份** `Grand` 子对象
- `Base2 b2;` 创建了另一个独立的对象，里面包含 **自己独占的另一份** `Grand` 子对象
这两份 `Grand` 是 **完全独立的内存空间**，互不影响。**`Base1 b1;` 和 `Base2 b2;` 是两个独立对象，各自拥有独立的 `Grand` 子对象，修改 `b1` 中的 `Grand` 不会影响 `b2` 中的 `Grand。`** 只有修改Derived实例化对象中的base1部分才会影响base2部分，因为在derived对象内部，base1成员和base2成员中的grand部分本质上是一个内存空间

**虚继承的“共享”特性只在“同一个最派生类对象内部”有效。**

- 不同对象之间，即使它们都是同一个中间基类类型，各自的虚基类也是独立的
- 共享是对象级别的，不是类级别的

```
// 顶层基类（虚基类）
class Grand {
public:
    int value;
    Grand() : value(10) {
        cout << "Grand 构造" << endl;
    }
};
// Base1 虚继承 Grand
class Base1 : virtual public Grand {
public:
    Base1() {
        cout << "Base1 构造" << endl;
    }
};
// Base2 虚继承 Grand
class Base2 : virtual public Grand {
public:
    Base2() {
        cout << "Base2 构造" << endl;
    }
};
// Derived 继承 Base1 和 Base2
class Derived : public Base1, public Base2 {
public:
    Derived() {
        cout << "Derived 构造" << endl;
    }
};
```
