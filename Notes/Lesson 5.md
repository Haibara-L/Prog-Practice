### 多态与虚函数
虚函数：在类的定义中,前面有virtual 关键字的成员函数 就是虚函数
- virtual 关键字只用在类定义里的函数声明中写，写函数体时不用
- 静态成员函数不能是虚函数
###### 多态的表现形式-1
派生类的指针可以赋给基类指针 
通过基类指针调用基类和派生类中的同名虚函数 时:
- 若该指针指向一个基类的对象,则被调用是基类的虚函数
- 若该指针指向一个派生类的对象,则被调用的是派生类的虚函数 
这种机制就叫做多态 
例如: CBase * p = &ODerived; p -> SomeVirtualFunction(); 
//即根据指针 /引用指向的对象类型来决定调用的函数

#### 多态的表现形式-2 
派生类的对象可以赋给基类引用 
通过该基类引用调用基类和派生类中的同名虚函数时:
- 若该引用引用的是一个基类的对象,那么被调用是基类的虚函数
- 若引用的是一个派生类的对象,那么被调用的是派生类的虚函数 
这种机制也叫多态 
例如： CBase & r = ODerived; r.SomeVirtualFunction()

在多层继承的 情况下, 从定义 virtual开始的派 生类中同名函数 均为虚函数, 无论在这些派生类 的同名函数中是 否显示加virtual。在派生类中可以加上override检查，既保证这个确实是重写了一个虚函数。当然加上virtual也可以

```
class Base {
public:
    virtual void func();
    virtual void foo();
};

class Derived : public Base {
public:
    void func() override;   // ✅ 明确是重写，编译器检查
    void foo() override;    // ✅ 同上
};
```
###### 多态的作用 
多态的实质 
- 父类定义共同接口,子类不同实现 
- 通过父类以相同的方式操作不同子类的行为 
在面向对象的程序设计中使用多态 
- 能够增强程序的可扩充性 
- 程序需要修改或增加功能的时候, 需要改动和增加 的代码较少


注意
1. 当父类的指针指向子类的对象时，通过该指针调用父类和子类同名的函数或变量，调用的都是父类的版本。这是因为普通函数和成员变量的访问遵循静态绑定（编译时绑定），由指针的静态类型（父类）决定，而不是由实际指向的对象的动态类型（子类）决定。
	 - **对于非虚函数**：编译器根据指针的静态类型确定调用哪个函数，父类指针只能看到父类中定义的函数，子类中的同名函数被隐藏。
	- **对于成员变量**：同样根据静态类型确定访问哪个变量，父类指针访问的是子类对象中属于父类部分的那份变量，子类中定义的同名变量位于子类扩展部分，父类指针无法访问到。
	- **对于虚函数**：则是例外，采用动态绑定，根据实际指向的对象类型决定调用哪个版本，从而实现多态。
2. 当子类和父类中具有同名的函数和变量，子类函数调用子类变量，父类函数调用父类变量，使用父类名::函数名调用的是父类变量，这是因为父类函数能够调用的只有父类作用域里面的变量，而子类函数调用的是子类变量，因为子类变量把父类变量隐藏了
3. 注意指针是通过->来调用的，引用是通过.来调用的,在填空时要注意传参是指针还是引用



#### 动态联编
一条函数调用语句在编译时无法确定调用哪个函数, 运行到该语句时才确定调用哪个函数, 这种机制叫动态联编。
动态联编的原因：父类指针调用哪一个子类的虚函数，在编译时无法确定，因此只有当程序运行到这里才能确定调用那个函数，因此需要动态联编
多态”的关键在于通过基类指针或引用调用一个虚函数时, 编译时不确定到底调用的是基类 还是派生类的函数, 运行时才确定. 这到底是怎么实现的呢
###### 虚函数指针与虚函数表
- 每个有虚函数的类(或有虚函数的类的派生类)都有一个 虚函数表,该类的任何对象中都放着虚函数表的指针，因此具有虚函数的内存占用会比可见的变量内存多出来4字节
- 虚函数表中列出了该类的虚函数地址 多出来的4个字节就是用来放虚函数表的地址
- 多态的函数调用语句被编译成一系列根据基类指针所指向的(或基类引用所引用的)对象中存放 的虚函数表的地址，==即赋给那个类的对象，存放的就是哪一个类的虚函数指针== → 在虚函数表中查找虚函数地址,并调用虚函数
###### 虚函数的访问权限
注意语法检查先于动态联编
访问权限检查是根据指针类型来的，而不是实际动态联编的结果
```
class Base { 
private: 
virtual void fun2() { cout << "Base::fun2()" << endl; } 
}; 
class Derived:public Base { 
public: 
virtual void fun2() { cout << "Derived:fun2()" << endl; }
}; 
Derived d; 
Base * pBase = & d; 
pBase-> fun2(); // 编译出错
```
- 编译出错: fun2()是Base的私有成员.即使运行到此时实际上调用的应该是Derived的公 有成员fun2()也不行,因为语法检查是不考虑运行结果的 
- 如果将Base中的private换成public,即使Derived中的fun2()是private的,编译依然 能通过,也能正确调用Derived::fun2()
#### 构造函数和析构函数中调用虚函数
在构造函数和析构函数中调用虚函数时: 
- 调用的函数是自己的类或基类中定义的函数, 不会等到运行时才决定调用自己的还是派生类的函数 
在普通成员函数中调用虚函数, 则是动态联编, 是多态

根本原因：
当执行派生类的构造函数时, 基类的构造函数会被调用 
- 尽管实际上正在创建一个派生类对象,但在基类的构造函数里调用相应函数 
- 在执行基类的构造函数时, 派生类成员变量还没有被初始化，比如虚函数表和虚函数指针都没有创建好
- 如果此时允许 调用派生类的虚函数,而这个函数访问了派生类自己的成员,那么程序就会去访问一片尚未初始化的内存,导致不可预知的错误 
- C++为了安全,规定在构造派生类时,对象的身份就是基类,而不是派生类
同理,析构时,先析构派生类部分，再析构基类部分 
- 当执行son析构函数时, 派生类部分已经被析构 
- 如果调用派生类里的虚函数,同样存在风险 
- 所以调用的是基类的虚函数

```
A * pa = new B();
pa->Func();
//发生了什么：pa 是一个基类指针，指向一个派生类 B 的对象。
底层逻辑：编译器看到 Func是虚函数，于是通过 pa 找到对象头部的 vptr（虚函数表指针）。此时 vptr指向的是 B 类的虚函数表。 
输出：B::Func。

long * p1 = (long * ) & a;  // p1 指向对象 a 的头部（即 a 的 vptr）
long * p2 = (long * ) pa;   // p2 指向对象 b 的头部（即 b 的 vptr）
* p2 = * p1;                // 关键：把 a 的 vptr 值 赋给了 b 的 vptr
//&a` 的头部是什么？** 是类 `A` 的虚函数表地址  
pa` 的头部是什么？** 原本是类 `B` 的虚函数表地址。   
赋值操作：** `*p2 = *p1` 实际上是强行修改了堆内存中那个 `B` 对象的“身份标签”。现在，这个 `B` 对象的 `vptr` 不再指向 B 的虚表，而是指向了 **A 的虚表**。

pa->Func();
- **发生了什么：** 再次调用 `pa->Func()`。   
- **底层逻辑：** 1. 编译器依然去查找 `pa` 指向对象的头部（vptr）。 2. 它发现 vptr 指向的是 **A 的虚表**（因为刚才被我们改了）。 3. 于是，它从 A 的虚表里找到了 `A::Func` 并执行
- **输出：** `A::Func`。
```

#### 虚析构函数
通过基类的指针删除派生类对象时 
- 通常情况下只调用基类的析构函数，而不会调用派生类的析构函数
但是直接删除一个派生类的对象时
- 要求先调用派生类的析构函数,然后调用基类的析构函数
解决办法:把基类的析构函数声明为virtual 
- 派生类的析构函数可以不声明为virtual
- 通过基类的指针删除派生类对象时, 首先调用派生类的析构函数,然后调用基类的析构函数 
一般来说, 一个类如果定义了虚函数, 则应该将析构函数也定义成虚函数 
==注意:不允许以虚函数作为构造函数==
构造函数的任务之一就是初始化内存空间, 包括设置虚函数表指针指向该类的虚函数表，否则会导致死循环
==执行派生类的析构函数，会引起调用基类的虚析构函数==
#### 纯虚函数与抽象类
纯虚函数: 没有函数体的虚函数
包含纯虚函数的类叫抽象类 
- 抽象类只能作为基类来派生新类使用,不能创建抽象类的对象 
- 抽象类的指针和引用可以指向由抽象类派生出来的类的对象
- A a ; // 错误, A 是抽象类,不能创建对象 
- A * pa ; // ok, 可以定义抽象类的指针和引用 
- pa = new A ; //错误, A 是抽象类, 不能创建对象

注意：
- 在抽象类的成员函数内可以调用纯虚函数 
- 但是在构造函数或析构函数内部不能调用纯虚函数。这是因为在构造函数或者析构函数中调用纯虚函数只能调用自己类或者基类中定义的函数，即调用虚析构函数本身，相当于调用一个不存在的函数，导致程序崩溃
- 如果一个类从抽象类派生而来,那么当且仅当它实现 了基类中的所有纯虚函数,它才能成为非抽象类
总结：
隐藏: 所访问的对象由 指针(或引用)的类型所 决定 
- 用派生类指针访问这类成 员时, 访问的是派生类中 定义的(同名)成员函数 
- 用基类指针访问这类成员 时, 访问的是基类的(同 名)成员函数, 即使它指 向的是一个派生类对象
虚函数: 所访问的对象 由指针(或引用)指向的 对象类型所决定 
- 若该指针(或引用)指向一 个基类的对象, 那么被调 用是基类的虚函数 
- 如果该指针(或引用)指向 一个派生类的对象, 那么 被调用的是派生类的虚函 数
