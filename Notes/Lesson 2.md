#### 构造函数&复制构造函数&析构函数
###### 构造函数
在程序未明确初始化时
- 全局基本类型变量自动初始化为0
- 局部基本类型变量随机赋值

构造函数：初始化对象
- 名字与类名相同
- 可以有参数
- 无返回值
- 作用：对对象进行初始化，如给成员变量赋值

三类构造函数
- 默认构造
- 有参构造（重写构造函数）
- 复制构造
- 一个类可以有多个构造函数，根据函数重载进行识别
==如果重写了有参构造，在创建对象时就必须传参数，此时默认构造不起作用，如果使用默认构造会报错==
==构造函数最好要用public，private无法直接用来初始化对象==

逗号表达式，依次从左向右执行，最后执行的结果是最右边的
`a=1,2`,执行结果最终a=1
`a=(1,2)`,最终a=2
###### 复制/拷贝构造函数
只有一个参数，即对同类对象的引用
```
X::X(X&)
X::X(const X&)
后者必须以常量引用为参数
```
- 默认的复制构造函数：完成复制功能
- 重写默认构造函数：执行自定义功能
- 如果重写了后者，那么前者不被执行
- 复制构造函数只有一个参数，只能是对同类对象的引用，不可以是同类对象

调用时机
- 用一个对象初始化同类的另一个对象，==对象之间用等号赋值并不会调用复制构造函数==
```
	Complex c2(c1)； 会调用
	Complex c2 = c1; //初始化语句, 非赋值语句，会调用
	Complex c2;
	c2=c1; //赋值语句，不会调用
```
- 函数传参的值拷贝：如果某函数有一个参数是类A 的对象, 那么该函 数被调用时, 类A的复制构造函数将被调用  ==由此可知为什么复制构造函数参数不可以是同类的对象，因为这样会导致构造函数被递归调用，无法停止==
- 函数返回值：如果函数的返回值是类A的对象时, 则函数返回时, A的复制构造函数被调用

常量引用参数的使用
- 目的：避免生成形参引发复制构造函数，导致内存开销大
- 使用常量引用作为参数
- 实参的值在函数中无法别改变

类型转换构造函数
- 目的：实现类型的自动转换
- 隐式转换：
```
	#include <iostream>
using namespace std;

class Complex {
public:
    double real, imag;
    
    Complex(int i) {  //类型转换构造函数
        cout << "IntConstructor called" << endl;
        real = i; 
        imag = 0;
    }
    
    Complex(double r, double i) {
        real = r; 
        imag = i;
    }
};

int main() {
    Complex c1(7, 8);
    Complex c2 = 12;
    c1 = 9;  // 9被自动转换成一个临时Complex对象
    cout << c1.real << "," << c1.imag << endl;
    return 0;
}
```
- 显示转换
```
	#include <iostream>
using namespace std;

class Complex {
public:
    double real, imag;
    
    explicit Complex(int i) {  //显式类型转换构造函数
        cout << "IntConstructor called" << endl;
        real = i;  
        imag = 0; 
    }
    
    Complex(double r, double i) {    
        real = r; 
        imag = i;   
    }
};

int main() {
    Complex c1(7, 8);
    Complex c2 = Complex(12);  //显式调用构造函数
    // c1 = 9;  // explicit关键字防止隐式自动转换，这行会出错
    c1 = Complex(9);  //显式调用构造函数，OK
    cout << c1.real << "," << c1.imag << endl;
    return 0;
}
```
###### 析构函数
- 名字与类名相同，前面加上一个~
- 一个类只有一个析构函数
- 对象消亡时，自动调用析构函数，通常用来释放手动分配的内存空间
- 对象数组生命期结束后，对象数组的每一个元素的析构函数都会被调用

常见调用时机
- delete语句执行，如果释放一个数组，应当用delete \[\]
- 在对象作为函数返回值返回值返回后调用，返回的实际上是一个临时变量，实际上会调用两次，一个是函数内的变量在函数结束后会被释放，调用一次析构函数，一次是临时变量赋值完毕后被释放会调用一次析构函数
- 临时对象消亡时调用
==全局变量先创建后消亡，后创建先消亡==
==new指针也会创建对应的对象，因此会调用构造函数，删除指针也会调用析构函数，但是new出来的对象如果不手动删除不会被消亡，也不会掉调用析构函数==

#### 成员函数重载与参数缺省
成员函数可以重载，也可以有默认参数，具体使用情况与普通函数相同
注意默认参数的函数要注意与其他函数之间不能有二义性
#### this指针
this指针指向当当前成员函数作用的对象，即调用这个成员函数的对象
使用情况
- 在成员函数参数与成员属性同名时，可以用this指针指向成员属性用来区分，通常情况下可以省略
- 可以`return *this`，返回值返回当前对象
#### 静态成员属性
在声明前加上static关键字的成员变量属性（变量/函数）
- 普通成员变量：每个对象有各自的一份
- 静态成员变量：一共只有一份，所有成员共享，sizeof运算符不会计算静态成员变量占据的空间。静态成员变量是储存在类空间中的

- 普通成员函数：必须具体作用于某个对象
- 静态成员函数：并不具体作用于某个对象，他只可以调用静态全局变量，不可以调用对象自身的非静态全局变量
访问静态成员的方式：
1. 通过：类名::成员名 的方式 CRectangle::PrintTotal();
2. 也可以和普通成员一样采取 
	- 对象名.成员名 r.PrintTotal(); 
	- 指针->成员名 CRectangle * p = &r 或者 p->PrintTotal(); 
	- 引用.成员名 CRectangle & ref = r 或者 int n = ref.nTotalNumber; 
上面这三种方式, 效果和 类名::成员名没区别
注意：静态成员变量不会属于某个特定对象且静态成员函数不会作用于某个特定对象

注意：
1. 静态成员变量本质上就是全局变量，即使一个对象都不存在，类的静态成员变量也存在
2. 静态成员函数本质上就是全局函数
3. ==必须在定义类定义的外面专门对静态成员变量进行声明，同行可以初始化; 否则编译能通过, 链接不能通过==，这是因为不可以再构造函数中对它进行初始化
4. 静态成员函数是全局函数，只可以调用一些全局的东西，==静态成员函数中,不能访问非静态成员变量，不能调用非静态成员函数==，这是因为他无法分清楚调用的到底是哪个对象中的变量
5. ==静态成员函数中不能使用 this 指针!== 因为静态成员函数并不具体作用与某个对象!  因此, 静态成员函数的真实的参数的个数, 就是程序中写出的参数个数
6. this指针并不是存在于对象的空间中的，不会占用对象的内存，**this 是作为成员函数的第一个隐藏参数被传递到函数中的。** 因此，它存在于 **函数的栈帧** 中（如果函数是普通调用）或者存在于 **寄存器** 中（经过编译器优化后）
7. 非静态成员函数可以调用静态的成员变量，成员函数
8. 设置静态成员这种机制的目的 
	- 将与某些类紧密相关的全局变量和函数写到类里面 
	- 看上去像一个整体 
	- 易于维护和理解
	- 例如，通过在构造函数和析构函数中修改静态成员变量，可以用来记录当前一共创建了多少个类成员
#### 内联函数
###### 内联函数
函数调用本身具有开销，如果函数只有几条语句，比如执行两个数相加，执行非常快，但是函数被反复执行，调用函数产生的开销比较大
在定义函数时，在返回值类型前面加上一个inline关键字，可以使函数称为内联函数，编译器处理内联函数调用的语句时，就会把整个函数代码块插入到调用语句中，而不会产生调用函数的语句，这样既可以实现对功能的封装，同时节省了简单函数的调用开销
###### 内联成员函数
对于类内定义，类外实现的函数，在类内定义的前面加上inline关键字，可以使它称为内联成员函数
对于类内定义和实现的函数，不用加上inline，会自动变成内联成员函数
通常来说，一般会把那些较小的函数修饰成为内联函数。但是注意，inline 对于编译器只是建议而非命令 编译器可能忽略内联请求 (对于过长的函数) 也可能自动内联未标记为 inline 的短小函数 (即使在类外部定义)

###### 常量对象
如果不希望某个对象的值被改变 → 则定义该对象的时候可以在前面加 const关键字
常量对象只能使用: 构造函数, 析构函数 和 有const说明的函数 (常量方法)

在类的成员函数说明后面加const关键字, 则该成员函数成为常量 成员函数 
常量成员函数内部不能改非静态属性的值,但可以改变静态成员属性的值， 也不能调用同类的非常量成员函数 (静态成员函数除外)
==理解：非静态成员函数可以修改非静态成员变量的值，因此常量对象不允许调用非静态成员函数==、

在定义常量成员函数和声明常量成员函数时都应该使用 const关键字，加上了const和没有加上const是两个不同的函数，二者形成重载关系，非常量对象默认调用非常量成员函数，常量对象只能调用常量成员函数

如果觉得传递一个对象效率太低 (导致复制构造函数调 用, 还费时间) 又不想传递指针, 又要确保实际参数的值不 能在函数中被修改, 那么可以使用: const T & 类型的参数，如果实在要调用，可以进行强制类型转换：`Sample & r = (Sample &) o; //必须强制类型转换`
###### mutable成员变量
可以在const成员函数中修改的成员变量，只要在类型前面加上mutable就可以了

==注意区分：静态成员函数不能访问，不能修改非静态成员变量，常量成员函数可以访问，不能修改不带mutable的非静态成员变量==
#### 成员对象和封闭类
成员对象: 一个类的成员变量是另一个类的对象
有成员对象的类叫 封闭类 (enclosing)

任何生成封闭类对象的语句, 都要让编译器明白 → 对象中的成员对象, 是如何初始化的 
具体的做法就是 通过封闭类的构造函数的初始化列表
定义封闭类的构造函数时, 添加初始化列表: 
类名::构造函数(参数表):成员变量1(参数表), 成员变量2(参数表), … { … } 
成员对象初始化列表中的参数 可以是：
1. 任意复杂的表达式
2. 函数 / 变量/ 表达式中的函数, 其中变量有定义
###### 封闭类的构造函数和析构函数的执行顺序
封闭类对象生成时 
- 先执行所有成员对象的构造函数  
- 然后才执行封闭类的构造函数 
对象成员的构造函数调用次序和对象成员在类中的声明次序一致， 与它们在成员初始化列表中出现的次序无关 
封闭类的对象消亡时
- 先执行封闭类的析构函数 
- 然后再执行成员对象的析构函数
- 次序和构造函数的调用次序相反

注意
1. 封闭类的对象, 如果是用默认复制构造函数初始化 → 它里面包含的成员对象, 也会用复制构造函数初始化
2. 初始化const 成员和引用成员时, 必须在成员初始化列表中进行，这是因为他们都是必须先确定，后续不可以改变的成员
3. ==核心思想就是，对于构成成员的关键部分，必须先于成员初始化，现有各个部分，才能构成整体==

#### 友元
###### 友元函数
在类的定义体内，使用关键字 `friend` 加上函数的原型声明。
**位置：** 可以放在类的 `public`、`protected` 或 `private` 部分，效果都是一样的，因为“朋友”关系不受访问限定符的影响。
1. 全局函数作友元：
```
class MyClass {
private:
    int secret;
public:
    MyClass(int s) : secret(s) {}
    // 声明全局函数 displaySecret 是这个类的朋友
    friend void displaySecret(const MyClass& obj);
};
```
2. 成员函数作友元
```
// 前向声明（告诉编译器有这么一个类）
class B;
class A {
private:
    int dataA;
public:
    A(int d) : dataA(d) {}
    // 声明 B 类的成员函数 showA 是 A 的朋友
    friend void B::showA(const A& a);
};
```
###### 友元类
在类定义中，使用 `friend` 关键字加上类名：
友元声明可以放在类的任何访问区域（public、protected、private），效果都一样：
```
class 类A {
    // ...
    friend class 类B;  // 声明类B是类A的友元
    // ...
};
```
友元的特点
1. 单向性：友元关系是**单向**的：
	- 如果B是A的友元，B可以访问A的私有成员
	- **但A不能访问B的私有成员**，除非A也被声明为B的友元
2. 不传递性：友元关系**不能传递**：
	- 如果B是A的友元，C是B的友元
	- **C不是A的友元**，除非显式声明
3.  不继承性：友元关系**不能被继承**：
	- 如果B是A的友元，B的派生类**不是**A的友元
