## 标准模板库STL
#### 基本概念
容器:可容纳各种数据类型的通⽤数据结构, 是类模板 
迭代器: 可⽤于依次存取容器中元素, 类似于指针 
- 普通的C++指针就是⼀种迭代器 
算法:⽤来操作容器中的元素的函数模板 
- sort()来对⼀个数组中的数据进⾏排序 
- find()来搜索⼀个数组中的对象 算法本⾝与他们操作的数据的类型⽆关, 因此他们可以在从简单数组到高度复杂容器的任何数据结构上使用
#### 容器
可以⽤于存放各种类型的数据 (基本类型的变量, 对象等) 的数据结构, 都是类模版, 分为三种
1. 顺序容器/序列容器(无序) vector, deque, list ：容器是非排序的, 元素的插⼊位置同元素的值⽆关
2. 关联容器/有序容器 set, multiset, map, multimap 
	- 元素是排序的 
	- 插⼊任何元素，都按相应的排序规则来确定其位置 
	- 在查找时具有⾮常好的性能 
	- 通常以平衡⼆叉树⽅式实现, 插⼊和检索的时间都是 O(logN)
3. 容器适配器 stack, queue, priority_queue、
	- 容器适配器, 就是将不适⽤的顺序容器(包括vector、deque 和list) 变得适⽤ 
	- 即通过封装某个顺序容器, 并重新组合该容器中包含的成员函数, 使其 满⾜某些特定场景的需要 
	- 容器适配器本质上还是容器, 只不过此容器模板类的实现, 利⽤了⼤量 其它基础容器模板类中已经写好的成员函数
前二者称为第一类容器，后者为第二类容器，后者之际上是对前者进一步包装形成的
- 对象被插⼊容器中时, 被插⼊的是对象的⼀个复制品
- 许多算法, ⽐如排序, 查找, 要求对容器中的元素进⾏⽐较, 有的容器本⾝就是排序的 
- 所以放⼊容器的对象所属的类, 往往还应该重载 == 和 < 运算符
###### vector
动态数组, 随机存取任何元素都能在常数时间完成，因为它是连续存储, 在尾端增删元素具有较佳的性能
size:当前有存储的数据大小
capacity：目前预留的大小
capacity>size，capacity会动态调整，当已有容量不够时会新开辟更大的内存空间，将旧的内存的数据复制过去
###### deque
双向队列, 元素在内存连续存放 随机存取任何元素都能在常数时间完成(但次于vector) 在两端增删元素具有较佳的性能(⼤部分情况下是常数时间）
deque内部实际上有两截，分别由两个指针管理，在第一个的头部和第二个的尾部具有预留的capacity空间，因此可以在常数时间内增删
###### list
双向链表, 元素在内存不连续存放 在任何位置增删元素都能在常数时间完成 不⽀持随机存取
###### stack：LIFO
项的有限序列, 并满⾜序列中被删除、检索和修改 的项只能是最近插⼊序列的项(栈顶的项) 即按照后进先出的原则
栈可以用来模拟递归
###### queue：FIFO
插⼊只可以在尾部进⾏, 删除、检索和修改只允许从头部进⾏，按照先进先出的原则
###### priority_queue
优先级队列 ,最高优先级元素总是第⼀个出列
###### 容器共有的成员函数
- 相当于按词典顺序⽐较两个容器的运算符: =, <, <=, >, >=, \==, != 
- empty: 判断容器中是否有元素 
- max_size: 容器中最多能装多少元素 
-  size:  容器中元素个数
- swap:交换两个容器的内容
###### 只在第一类容器中的成员函数
- begin 返回指向容器中第⼀个元素的迭代器 
- end 返回指向容器中最后⼀个元素后⾯的位置的迭代器 
- rbegin 返回指向容器中最后⼀个元素的迭代器 
- rend 返回指向容器中第⼀个元素前⾯的位置的迭代器 
- erase 从容器中删除⼀个或⼏个元素 
- clear 从容器中删除所有元素
第二类容器访问有限制，因此不具有这些成员函数
###### 顺序容器的常用成员函数
- front 返回容器中第⼀个元素的引⽤ 
- back 返回容器中最后⼀个元素的引⽤
- push_back 在容器末尾增加新元素 
- pop_back 删除容器末尾的元素 
- erase  删除迭代器指向的元素(可能会使该迭代器失效),  或删除⼀个区间, 返回被删除元素后⾯的那个元素的迭代器
###### 迭代器
⽤于指向第⼀类容器中的元素,有const和⾮const两种 
通过迭代器可以读取它指向的元素 通过⾮const迭代器还能修改其指向的元素 
迭代器⽤法和指针类似
```
//定义⼀个容器类的迭代器的⽅法： 
容器类名::iterator变量名; 
容器类名::const_iterator变量名; 
//访问⼀个迭代器指向的元素： 
*迭代器变量名
```
迭代器上可以执⾏ ++ 操作, 以使其指向容器中的下⼀个元素 。
如果迭代器到达了容器中的最后⼀个元素的后⾯ 
- 迭代器变成 past‐the‐end 值 
- 此时再使⽤它，就会出错 
- 类似于使⽤ NULL 或未初始化的指针⼀样
注意一种特殊的迭代器：反向迭代器
```
vector::reverse_iterator r; //反向迭代器 
for( r = v.rbegin(); r != v.rend(); r++ )  
cout << * r << ","
```
反向迭代器的++相当于向前移动

不同容器上⽀持的迭代器功能强弱有所不同 
功能强弱, 决定了该容器是否⽀持STL中的某种算法 
- 只有第⼀类容器能⽤迭代器遍历 
- 排序算法需要通过随机迭代器来访问容器中的元素，那么有的容器就不⽀持排序算法

STL中的迭代器按功能由弱到强分为5种 
1. 输⼊:提供对数据的只读访问，输出:提供对数据的只写访问 
2. 正向:提供读写操作, 并能⼀次⼀个地向前推进迭代器 
3. 双向:提供读写操作, 并能⼀次⼀个地向前和向后移动 
4. 随机访问: 提供读写操作, 并能在数据中随机移动，因此要求内存空间连续，故必须是数组类型才可以
编号⼤的迭代器拥有编号⼩的迭代器的所有功能, 能当作编号⼩ 的迭代器使⽤
- 所有迭代器: ++p, p++ 
- 输⼊迭代器: * p, p = p1, p == p1, p != p1 
- 输出迭代器:\*p, p=p1 
- 正向迭代器: 上⾯全部 
- 双向迭代器: 上⾯全部,‐‐p, p‐‐, 
- 随机访问迭代器（就相当于指针了）: 上⾯全部, 以及: 
	- 移动i个单元: p += i, p‐= i, p + i, p– i 
	- ⼤于/⼩于⽐较: p < p1, p <= p1, p > p1, p>= p1 
	- 数组下标p\[i]: p后⾯的第i个元素的引⽤‘



#### 算法
- STL提供能在各种容器中通⽤的算法 
- 算法就是⼀个个函数模板
- 算法通过迭代器来操纵容器中的元素。  
- 许多算法可以对容器中的⼀个局部区间进⾏操作, 因此需要两个参数, ⼀个是起始元素的迭代器, ⼀个是终⽌元素的后⾯⼀个元素的迭代器
- 有的算法返回⼀个迭代器. ⽐如 find() 算法, 在容器中查找⼀个元素, 并返回⼀个指向该元素的迭代器 
- 算法可以处理容器，也可以处理C语⾔普通数组
- 大部分在algorithm中定义，部分在numeric中定义

#### 函数对象与流迭代器

1. 函数对象 (Function Objects / Functors)

函数对象是 STL 算法实现高度可复用性的核心机制，它允许将具体的“子操作”（如排序规则、累加逻辑）作为参数传递给算法 。

- **核心定义**：
    
    - **函数对象类**：一个重载了 `operator()` 成员函数的类 。
        
    - **函数对象**：该类的实例化对象 。其使用方式在语法上与传统的函数调用完全一致，但实质上是触发了类成员函数的调用 。
        
- **设计目的**：
    
    - **参数化子操作**：为了使 STL 算法能够复用。例如，`sort` 算法可以通过不同的函数对象来切换“升序”或“降序”规则 。
        
    - **状态保持**：与普通函数不同，函数对象可以拥有成员变量（如私有属性），从而在多次调用间保持特定的状态信息 。
        
- **应用实例：`accumulate` 算法** ：
    
    - **普通版本**：仅实现简单的数值累加 。
        
    - **函数对象版本**：接受一个二元操作符（可以是函数名、函数指针或函数对象），对区间内的元素执行自定义计算逻辑 。
        
    - **源码逻辑**：`__init = __binary_op(__init, *__first);` 。
        

2. ostream_iterator（输出流迭代器）

`ostream_iterator` 是一个模板类，属于**输出迭代器**的一种，其作用是将“向迭代器赋值”的动作转化为“向输出流写入数据”的动作 。

- **基本构造与用法**：
    
    - **定义方式**：`ostream_iterator<T> output(cout, "sep");` 。
        
    - **参数含义**：第一个参数绑定输出流（如 `cout`），第二个参数是可选的分隔符字符串，每次写入数据后会自动插入该分隔符 。
        
- **与算法的配合（以 `copy` 为例）**：
    
    - 在执行 `copy(v.begin(), v.end(), output);` 时，算法内部会循环执行 `*output = *it;` 。
        
    - 这一赋值动作直接导致数据流向绑定的输出端（如屏幕），实现了“一句话完成容器打印”的功能 。
        
- **内部实现原理（伪代码分析）** ： 为了伪装成迭代器以适配算法模板，`ostream_iterator` 必须重载以下运算符 ：
    
    - **`operator=`**：**最核心的重载**。输出操作实际上是在赋值运算符中完成的 。
        
    - **`operator*`**：通常直接返回 `*this` 。
        
    - **`operator++`**：仅需提供空定义以满足算法中 `++x` 的语法要求，实际不执行任何位置移动 。
        

3. 补充：istream_iterator（输入流迭代器）

- 用于从输入流中读取数据 。
    
- 通过 `*inputInt` 触发读取操作，通过 `inputInt++` 移动到流中的下一个数据元素 。
